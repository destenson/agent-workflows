# CHANGELOG.md — change record

Append-only, newest first. One entry per change made to a live system. Recorded as the change is made (per the standing rules). This is the record that answers "what changed?" when something that worked last week stops working.

This is operator-facing change history for the fleet — distinct from any application or release changelog.
