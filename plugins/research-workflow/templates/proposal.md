# Proposal — {project title}

The durable framing of what is being tested and how. A starting stub — the `generate-research-proposal` (and feature/product/project) skills produce the full proposal; this artifact holds the committed core that the rest of the work is judged against.

- **Hypotheses**: the specific, testable claims. Each should have a way to be wrong.
- **Approach**: how the question gets answered — the method, at the level of detail a reader needs to judge whether it can.
- **Success / decision criteria**: pre-committed. The metric(s), the baseline(s) to beat, and the rule that says ship / iterate / kill. Set before any result is seen.
- **Risks & unknowns**: what could make this not work, and what is assumed but not yet verified (cross-reference the assumptions, if tracked separately).
- **Out of scope**: what this proposal explicitly does not attempt.
