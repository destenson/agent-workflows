--- SESSION PROBE (research-workflow) ---

The artifacts above are this project's research memory (commonly the abstract, proposal, experiment notes, and results — the set varies with the work). If none were shown, this project is not yet initialized — run research-init skill before doing research work.

Before running an experiment or writing analysis, summarize:
1. The current research question and the live hypotheses — what is being tested and what would confirm or falsify it.
2. What has already been decided (methods, scope, ship/iterate/kill calls) and what has already been tried and falsified — so a dead end is not re-walked.
3. The pre-committed criteria for the current task: the metric, baseline, and stopping/decision rule. State these before seeing results, not after.
4. The scope of the current task and which artifact it produces or updates.

Do not draw a conclusion before the criteria it is judged against are stated. The most common research self-deception is reading the success bar off the result; pre-committing the bar is what this summary protects.
